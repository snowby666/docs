## API Versioning

We use semantic versioning for our API:
- **Major version** (X.0.0): Breaking changes that require code updates
- **Minor version** (X.Y.0): New features that are backward compatible  
- **Patch version** (X.Y.Z): Bug fixes and improvements

### Version Support
- **Current version**: v1.2.0 (fully supported)
- **Previous version**: v1.1.0 (security updates only)
- **Legacy versions**: v1.0.x (deprecated, sunset in 6 months)

## Breaking Changes Policy

We avoid breaking changes whenever possible. When necessary:
- 90-day advance notice via email and documentation
- Migration guides provided
- Legacy endpoint support during transition period
- Developer support for migration assistance

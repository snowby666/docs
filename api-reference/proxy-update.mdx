---
title: "Update Proxy Key"
api: "POST https://api.electronhub.ai/v1/auth/proxy/update"
description: "Update an existing proxy key's configuration"
openapi: "POST /auth/proxy/update"
---

Update the configuration of a proxy key you own. The proxy key to update is now identified by the `proxy_key` field in the request body so the URL is static and can be exact-matched at the edge (Cloudflare, etc.), making it immune to path-traversal-append DDoS abuse.

The legacy `POST /v1/auth/proxy/update/{proxy_key}` still works for backward compatibility but is discouraged for new integrations.

<RequestExample>

```bash cURL
curl https://api.electronhub.ai/v1/auth/proxy/update \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $ELECTRONHUB_API_KEY" \
  -d '{
    "proxy_key": "ek-proxy-1234567890abcdef",
    "name": "Updated Production Key",
    "expires_at": 1767225600,
    "allocated_ammount": 150.0,
    "model_whitelist": ["gpt-4o", "claude-3-5-sonnet-20241022", "dall-e-3"],
    "ip_whitelist": ["192.168.1.0/24"]
  }'
```

```python Python
import httpx
import time

        "Content-Type": "application/json",
    },
    json={
        "proxy_key": "ek-proxy-1234567890abcdef",
        "name": "Updated Production Key",
        "expires_at": expires_at,
        "allocated_ammount": 150.0,
        "model_whitelist": ["gpt-4o", "claude-3-5-sonnet-20241022", "dall-e-3"],
        "ip_whitelist": ["192.168.1.0/24"],
    },
)

result = response.json()
print(f"Update result: {result['message']}")
```

## Request Body

<ParamField body="proxy_key" type="string" required>
  The proxy key to update (starts with `ek-proxy-`). Must be a string between 1 and 256 characters and must be a proxy key you own &mdash; cross-account updates return `403 Forbidden`.
</ParamField>

<ParamField body="name" type="string" required>
  Updated human-readable name for the proxy key (max 25 characters).
</ParamField>

<ParamField body="expires_at" type="integer" required>
  New expiration timestamp in seconds since Unix epoch. Use `-1` for no expiration.
</ParamField>

<ParamField body="allocated_ammount" type="number" required>
  New credit amount to allocate to this proxy key. Must be positive and cannot exceed your tier's per-key cap (default $10,000).
</ParamField>

<ParamField body="limit_type" type="string" default="daily">
  Spend window for the allocation. One of `"daily"`, `"weekly"`, or `"monthly"`.
</ParamField>

<ParamField body="model_whitelist" type="array">
  Updated list of model IDs this key can access. Leave empty or omit to allow all models.
</ParamField>

<ParamField body="ip_whitelist" type="array">
  Updated list of IP addresses or CIDR blocks allowed to use this key. Leave empty or omit to allow all IPs.
</ParamField>

## Important Notes

<Warning>
**Credit Changes**: When increasing `allocated_ammount`, the difference is deducted from your account. When decreasing, unused credits are returned.
</Warning>

<Warning>
**Active Requests**: Changes take effect immediately. Any in-flight requests using the proxy key complete with the old settings.
</Warning>

<Info>
**Usage Preservation**: `used_ammount` is preserved across updates &mdash; only the allocation limit changes.
</Info>

## Error Codes

<ResponseField name="400" type="error">
  **Bad Request** &mdash; invalid input data or validation failed.
</ResponseField>

<ResponseField name="401" type="error">
  **Unauthorized** &mdash; missing, malformed, or invalid `Authorization` header. JWT tokens are not accepted on this endpoint.
</ResponseField>

<ResponseField name="403" type="error">
  **Forbidden** &mdash; the proxy key exists but is owned by a different account.
</ResponseField>

<ResponseField name="404" type="error">
  **Not Found** &mdash; no proxy key with that value exists.
</ResponseField>

<ResponseField name="400" type="error">
  **Bad Request** &mdash; the request body is missing `proxy_key` or any required field, or violates the length constraint (`1 <= len(proxy_key) <= 256`). Returned as the standard [error envelope](/getting-started/errors) (`{"error": {...}}`).
</ResponseField>

<ResponseField name="429" type="error">
  **Rate Limited** &mdash; too many requests, please slow down.
</ResponseField>

## Notes

- Requires an **API key** (`ek-...`) in the `Authorization` header. JWT tokens are intentionally rejected.
- Proxy keys themselves (`ek-proxy-...`) cannot call this endpoint &mdash; only the parent account that owns the proxy key can update it.
